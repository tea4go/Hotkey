# Linux 全局快捷键工具 — 需求文档

## 一、项目背景

Linux 桌面缺少一个开箱即用、跨显示服务器、配置即用的全局快捷键工具。现有方案存在以下问题：

- **sxhkd、xbindkeys：** 仅支持 X11，Wayland 下完全失效。
- **GNOME / KDE 自带快捷键：** 强绑定桌面环境，跨 DE 不通用，且配置入口分散。
- **AutoKey：** Python 实现，依赖较重，X11 限定。
- **keyd：** ��向键位重映射，做命令触发需额外配置。

本项目目标：用 Go 实现一个**轻量、单二进制、X11 环境下**的全局快捷键守护进程，专注于「按键 → 执行命令」这一核心场景，支持 NoMachine 等远程桌面。

## 二、目标用户

- Linux 桌面用户，希望用统一配置管理快捷键
- 系统管理员，需要在无 GUI 环境下绑定运维命令
- 重度键盘用户，希望脱离桌面环境的快捷键限制

## 三、功能需求

### 3.1 核心功能

| 编号 | 功能 | 优先级 |
|------|------|--------|
| F-01 | 通过 YAML 配置文件定义快捷键与命令的映射 | P0 |
| F-02 | 支持组合键（Ctrl / Alt / Shift / Super + 单键） | P0 |
| F-03 | 守护进程模式后台运行 | P0 |
| F-04 | 监听所有键盘设备，新设备热插拔自动识别 | P0 |
| F-05 | 配置文件热重载（收到 `SIGHUP` 信号重新加载） | P1 |
| F-06 | 支持多键序列（如 `Ctrl+K, Ctrl+B` 这种 chord） | P2 |
| F-07 | 日志记录触发的快捷键和命令执行结果 | P1 |

### 3.2 命令执行

| 编号 | 功能 | 优先级 |
|------|------|--------|
| E-01 | 执行任意 Shell 命令 | P0 |
| E-02 | 支持以指定用户身份运行命令（避免命令全部以 root 启动） | P0 |
| E-03 | 命令异步执行，不阻塞热键监听 | P0 |
| E-04 | 支持配置环境变量（如 `DISPLAY`、`WAYLAND_DISPLAY`） | P0 |
| E-05 | 配置命令的工作目录 | P1 |

### 3.3 运维相关

| 编号 | 功能 | 优先级 |
|------|------|--------|
| O-01 | 提供 systemd service 文件，开机自启 | P0 |
| O-02 | `--check` 子命令校验配置文件合法性 | P1 |
| O-03 | `--list` 子命令列出当前所有热键绑定 | P1 |
| O-04 | 配置文件解析错误时给出具体行号和错误原因 | P1 |

## 四、非功能需求

### 4.1 性能

- **按键延迟：** 从按键到命令启动，端到端延迟 < 20 ms
- **CPU 占用：** 空闲时 CPU 使用率 < 0.1%
- **内存占用：** 常驻内存 < 20 MB

### 4.2 兼容性

- **显示服务器：** X11（通过 XInput2 扩展监听按键）
- **不支持：** Wayland-only 环境、纯 TTY 无 X 环境
- **远程桌面：** 支持 NoMachine、VNC、RDP 等通过 X11 注入键盘事件的方案
- **发行版：** Ubuntu 20.04+、Debian 11+、Fedora 35+、Arch Linux

### 4.3 安全性

- 守护进程以 root 运行，但**触发的命令默认降权**到指定用户
- 配置文件权限校验，非 root 可写时拒绝加载
- 不执行任意来源的命令，仅执行配置文件中预定义的命令

### 4.4 可维护性

- 单二进制部署，无外部依赖
- 配置文件支持注释，可读性强
- 提供清晰的错误日志和调试日志（`-v` 选项开启）

## 五、约束条件

| 维度 | 约束 |
|------|------|
| 编程语言 | Go 1.21+ |
| 运行权限 | root（或属于 `input` 组） |
| 部署形态 | 单二进制 + 配置文件 + systemd unit |
| 外部依赖 | 不依赖 X11 / Wayland 库，不依赖 CGO |
| 配置格式 | YAML |

## 六、不在本期范围

明确**不做**以下功能，避免范围膨胀：

- 鼠标手势、鼠标按键绑定
- 字符串展开 / 文本替换（这是 espresso、AutoKey 的领域）
- 键位重映射（这是 keyd 的领域）
- 图形化配置界面
- 跨平台支持（macOS / Windows）

## 七、验收标准

1. 配置 5 个常用快捷键（终端、文件管理器、锁屏、截图、浏览器），均可正常触发
2. 在 X11 和 Wayland 两套环境下分别验证可用
3. 热插拔一个外接键盘，新键盘上的快捷键可被识别
4. 修改配置文件后 `kill -HUP <pid>` 立即生效
5. 守护进程稳定运行 7 天无内存泄漏、无崩溃
